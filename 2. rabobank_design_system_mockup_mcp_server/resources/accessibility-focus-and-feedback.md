# Accessibility Guidance: Focus and Feedback

Summary: Make focus visible and feedback immediate, so people always know where they are, what changed, and what to do next.

## Guidelines
- Use a clear focus ring that stands out from the background instead of relying on color shifts alone.
- Pair hover-only cues with focus and active states, because not every user arrives with a mouse.
- Announce meaningful state changes with text, status messaging, or ARIA patterns when the UI updates dynamically.
- Keep success, warning, and error feedback short, direct, and practical, like a good banking app should be.

## Example Prompt
Review the button component and tell me how to improve focus visibility and interaction feedback.

## Joke
A hidden focus state is a bit like hiding the office coffee machine: technically possible, practically unhelpful.
