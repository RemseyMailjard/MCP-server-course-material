# Accessibility Guidance: Forms and Actions

Summary: Keep forms and calls to action simple, well-labeled, and forgiving, so users can complete tasks with confidence rather than guesswork.

## Guidelines
- Match every field with a visible label and keep placeholder text as support, not as the only instruction.
- Explain errors in plain language and place them close to the field or action that needs attention.
- Use button labels that describe the result, such as Continue to payment or Save profile details.
- Give people enough time for decisions and avoid interaction patterns that depend on precision clicking.

## Example Prompt
Show me accessibility notes for action labels and form feedback in the navbar call-to-action flow.

## Joke
A vague submit button is like a mystery transfer: exciting for exactly the wrong reasons.
