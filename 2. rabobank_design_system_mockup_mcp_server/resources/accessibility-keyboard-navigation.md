# Accessibility Guidance: Keyboard Navigation

Summary: Make every important action reachable without a mouse, so the flow feels steady, clear, and reliable from the first Tab to the last button.

## Guidelines
- Start with native elements like button, a, input, and select before reaching for custom interaction patterns.
- Keep the tab order aligned with the visual order, so the interface moves the way people expect.
- Avoid keyboard traps; users should always be able to move in and out of overlays, menus, and panels.
- Give skip links or quick-jump patterns to longer pages when repeated navigation starts to feel like extra admin.

## Example Prompt
Give me keyboard navigation notes for the feature-panel component in a calm Rabobank tone.

## Joke
Accessibility and keyboards have one thing in common: both work best when nothing gets in the way.
